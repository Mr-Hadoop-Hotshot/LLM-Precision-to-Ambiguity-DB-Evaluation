1.1 Missing Value Treatment:

price

There are numerous different products in this dataframe. Therefore, replacing NaN values using the mean does not make sense. Additionally, finding the local mean for each category also may not provide accurate information, as significant price variations between brands are observed. Incorporating brand information into a groupby operation similarly does not make sense, as the description reveals unique product characteristics. For example, products of the same brand and category, such as 'Gasket', can have multiple SKUs with different price ranges. Even within the category [[Appliances, Parts & Accessories, Refrigerator Parts & Accessories, Doors]], prices can range from approximately $$ 30 to $ 84.

import pandas as pd
from openai import OpenAI
from pinecone import Pinecone, ServerlessSpec

#  OpenAI client initialisation:
OPEN_AI_API_KEY = '****'
openai_client = OpenAI(api_key=OPEN_AI_API_KEY)

# Pinecone client instance initialisation:
PINECONE_API_KEY = '***'
pc = Pinecone(api_key=PINECONE_API_KEY)

# Index configuration
INDEX_NAME = 'amazon-meta-review-data1'
DIMENSION = 1536  

# Index Creation:
pc.create_index(
    name=INDEX_NAME, 
    dimension=DIMENSION, 
    metric='euclidean',
    spec=ServerlessSpec(
        cloud='aws',
        region='us-east-1'
    )
)
index = pc.Index(INDEX_NAME)

# Vector embedding conversion:
def get_embedding(text):
    response = openai_client.embeddings.create(input=[text], engine='text-embedding-ada-002')
    return response['data'][0]['embedding']

# Preprocessing function:
def prepare_data(df):
    upsert_data = []
    for i, row in df.iterrows():
        embedding = get_embedding(row['reviewText'])
        upsert_data.append((str(i), embedding, {
            'reviewerID': row['reviewerID'],
            'asin': row['asin'],
            'reviewerName': row['reviewerName'],
            'helpful': row['helpful'],
            'overall': row['overall'],
            'summary': row['summary'],
            'reviewTime': row['reviewTime'],
            'price': row['price'],
            'imUrl': row['imUrl'],
            'description': row['description'],
            'categories': row['categories'],
            'title': row['title'],
            'brand': row['brand'],
            'related': row['related'],
            'salesRank': row['salesRank']
        }))
    return upsert_data

# Function to upsert data in batches:
def upsert_in_batches(df, batch_size=1000):
    total_rows = len(df)
    for start in range(0, total_rows, batch_size):
        end = min(start + batch_size, total_rows)
        batch_df = df[start:end]
        batch_data = prepare_data(batch_df)
        index.upsert(vectors=batch_data)
